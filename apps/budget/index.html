Emotion-based Budget App

Personal Project · UX / API

감정을 먼저 기록하는 가계부 실험. 숫자가 아니라 반응을 중심으로 지출을 관찰한다.

Overview

이 프로젝트는 기존 가계부가 가진 복잡성과 지속성 부족 문제에서 출발했다. 문제는 의지가 아니라 구조라고 판단했다.

Why I Built This

대부분의 가계부는 오래 쓰기 어렵다. 입력 피로도보다 더 큰 문제는, 지출이 감정과 분리되어 있다는 점이었다.

Core Ideas

  • 금액보다 감정을 먼저 기록
  • 분류보다 반응 중심 설계
  • 초등학생도 쓸 수 있는 UI

Tech Stack

  • Frontend: HTML / CSS / Vanilla JS
  • Backend: API 기반 서버

Next Steps

감정 데이터의 흐름과 반복 패턴을 시각화하는 방향으로 확장할 예정이다.
