오늘 배운 것

MySQL + DBeaver 설치하고 기본 쿼리 연습했다.

환경 세팅

  • Homebrew로 MySQL 설치
  • DBeaver Community 설치
  • CSV 파일 인코딩 변환 (CP949 → UTF-8)

따옴표 국룰

따옴표 용도 예시
` ` 백틱 테이블명, 컬럼명 (한글/예약어) `상품`, `order`
' ' 작은따옴표 문자열 값 '홍길동', 'usa'

기본 쿼리

-- 데이터베이스 선택
USE 데이터베이스명;

-- 테이블 목록
SHOW TABLES;

-- 테이블 구조
DESC 테이블명;

-- 전체 조회
SELECT * FROM 테이블명;

-- 특정 컬럼 조회
SELECT name, age FROM 테이블명;

조건 검색 (WHERE)

-- 같다
SELECT * FROM sample21 WHERE country = 'usa';

-- 같지 않다
SELECT * FROM sample21 WHERE name != '홍길동';

-- IS NOT은 NULL 전용!
SELECT * FROM sample21 WHERE name IS NOT NULL;

LIKE 패턴 검색

-- ~로 시작
SELECT * FROM customer WHERE name LIKE '홍%';

-- ~로 끝남
SELECT * FROM customer WHERE email LIKE '%gmail.com';

-- ~포함
SELECT * FROM customer WHERE name LIKE '%길%';

기타 유용한 것들

-- 정렬
SELECT * FROM sample21 ORDER BY name DESC;

-- 개수 제한
SELECT * FROM sample21 LIMIT 10;

-- 중복 제거
SELECT DISTINCT country FROM sample21;

-- 범위
SELECT * FROM sample21 WHERE age BETWEEN 20 AND 30;

-- 목록 중 하나
SELECT * FROM sample21 WHERE country IN ('usa', 'korea');

-- 집계 함수
SELECT COUNT(*) FROM sample21;
SELECT SUM(price), AVG(price) FROM sample21;

GROUP BY

-- 그룹별 집계
SELECT country, COUNT(*) FROM sample21 GROUP BY country;

-- 그룹 조건 (HAVING)
SELECT country, COUNT(*)
FROM sample21
GROUP BY country
HAVING COUNT(*) > 5;

삽질 기록

  • SQLite로 연결해놓고 왜 안 되지 함 → MySQL로 다시 연결
  • 한글 깨짐 → CSV 인코딩 UTF-8로 변환
  • 백틱 치는데 ₩ 나옴 → 영어 키보드로 전환 or Option + ₩
  • IS NOT 썼다가 에러 → 값 비교는 != 써야 함
  • 따옴표 안 감쌌다가 에러 → 문자열은 무조건 '값'

결론: SQL은 쉬운데 문법이 깐깐하다.
