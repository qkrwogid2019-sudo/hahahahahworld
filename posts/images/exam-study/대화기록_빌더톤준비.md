## 빌더톤 신청서 내용 (작성 완료)

### 해결하고자 했던 문제 (Problem Definition) - 300자

신축 아파트 입주 전 사전점검, 전월세 계약 전 집 상태 확인은 필수지만,
대부분의 사람들은 "뭘 봐야 하는지" 모릅니다.

전문 점검 업체를 부르면 10~30만원, 하지만 모든 사람이 그 비용을 감당하기 어렵습니다.
유튜브나 블로그를 찾아봐도 "균열인지 그냥 무늬인지" 판단이 어렵고,
발견해도 "이게 심각한 건지, 수리비는 얼마인지" 알 수 없습니다.

결국 하자를 놓치고 입주 후 뒤늦게 발견하거나,
전월세 퇴거 시 원상복구 비용 분쟁으로 이어집니다.
비전문가도 쉽게 하자를 찾고, 심각도를 판단할 수 있는 도구가 필요합니다.

### 서비스 소개 및 주요 기능 (Solution) - 500자

하자탐정은 AI 기반 건물 하자 사전점검 도우미입니다.

[주요 기능]

1. 공간별 체크리스트
- 현관, 거실, 주방, 화장실, 방, 베란다 6개 영역
- 각 공간에서 반드시 확인해야 할 항목을 체크리스트로 제공
- 점검 누락 방지

2. 사진 촬영 + 마킹
- 문제 의심 부분을 사진으로 찍고
- 화면에서 직접 동그라미를 그려 표시
- 여러 문제를 번호로 구분 (1, 2, 3...)

3. AI 하자 분석
- Claude Vision API를 활용한 이미지 분석
- 표시된 각 부분에 대해 개별 분석
- 문제 유형 (균열/곰팡이/누수 등)
- 심각도 (경미/보통/심각)
- 원인 추정 및 대응 방법
- 예상 수리 비용
- 긴급도 판단

4. 결과 리포트
- 분석 결과를 한눈에 정리
- 심각도별 색상 구분 (초록/주황/빨강)
- 종합 조언 및 우선순위 제안

일반인도 전문가 수준의 점검이 가능하며,
점검 업체 비용(10~30만원)을 절감할 수 있습니다.

### 핵심 기술 및 AI 모델 (Tech Stack)

- Frontend: HTML5, CSS3, Vanilla JavaScript
- Backend: Vercel Serverless Functions
- AI: Claude API (claude-sonnet-4-20250514, Vision)
- Deploy: Vercel
- 기타: Canvas API (이미지 마킹), FileReader API (이미지 처리)

### 기술적 챌린지 및 해결 과정 - 500자

[챌린지 1: 모바일 캔버스 터치 드로잉]
문제: 모바일에서 손가락으로 원을 그릴 때 화면 스크롤과 충돌
해결: touch-action: none CSS 속성과 preventDefault()로 터치 이벤트 분리.
터치 좌표를 캔버스 비율에 맞게 변환하는 getPos() 함수 구현.

[챌린지 2: 다중 하자 개별 분석]
문제: 사용자가 여러 곳에 동그라미를 그렸을 때 AI가 뭉뚱그려 답변
해결: 동그라미에 번호(1,2,3)를 표시하고, 프롬프트에 "각 번호별로 개별 분석" 지시.
출력 형식을 HTML 템플릿으로 구조화하여 일관된 결과 보장.

[챌린지 3: 벽지 무늬 vs 실제 균열 오인식]
문제: AI가 벽지 패턴을 균열로 잘못 판단하는 케이스 발생
해결: 프롬프트에 "추론이 아닌 실제로 보이는 것 기반 판단" 지침 추가.
"물 얼룩, 변색, 곰팡이 등 시각적 증거를 놓치지 말 것" 명시.
추후 실제 하자 이미지 데이터셋으로 파인튜닝 예정. (만장 이상 보유)

[챌린지 4: 이미지 용량 최적화]
문제: 고해상도 사진 전송 시 API 응답 지연
해결: canvas.toDataURL()에서 JPEG 품질을 0.8로 설정하여 용량 절감.
